# Contributing Guide

## Contribution intent

This repository should grow in a disciplined way so that instructional content, case studies, visual assets, and supporting notes remain coherent.

## Good contribution types

- module writeups
- case study pages
- glossary additions
- diagrams and infographics
- lab guides
- instructor notes
- reference improvements

## Contribution rules

- write in clear Markdown
- use meaningful headings
- define complex terms when first introduced
- connect technical descriptions to operational relevance
- keep examples accurate and attributable
- prefer structured updates over random note dumps

## Before submitting

Check that your addition:

- fits the repo structure
- uses consistent terminology
- adds value for learners or instructors
- does not duplicate an existing page
