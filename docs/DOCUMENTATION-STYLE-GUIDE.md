# Documentation Style Guide

## Tone

Use a professional, direct, instructional tone. The writing should be clear enough for learners but precise enough for practitioners.

## Page structure

Most docs pages should follow this pattern:

1. Title  
2. Short overview  
3. Core concepts or sections  
4. Practical relevance  
5. Teaching or usage note where relevant

## Formatting rules

- use sentence-case headings where possible
- prefer short paragraphs
- use bullet lists for scanability
- define specialist terminology clearly
- avoid unnecessary filler language

## Content rule

Do not describe a cyber concept only technically when its operational meaning is the actual learning objective.
