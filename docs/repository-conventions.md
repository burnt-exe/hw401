# Repository Conventions

## Naming

Use uppercase hyphenated names for major Markdown docs where appropriate, especially in structured documentation folders.

## Folder expectations

- `docs/course/` for formal course pages
- `docs/frameworks/` for conceptual models
- `docs/instructor/` for delivery support
- `docs/student/` for learner-facing material
- `docs/references/` for support material
- `docs/repo/` for repository governance and process

## Content expectation

Every page should answer one clear question. If a page tries to answer too many questions, split it.

## Future-ready approach

Write pages so they can later be reused in:

- GitHub Pages
- MkDocs
- instructor manuals
- student handbooks
- LMS import pipelines
